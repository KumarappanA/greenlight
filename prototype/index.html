No public object storage hard block

POL-007 · v3 · owned by Platform Security · last edited May 2026

The rule

S3 buckets (and equivalent object stores) are never publicly readable or writable, in any environment. Public distribution of static content goes through a CDN with origin access control, so the bucket itself stays private.

Why it exists

Public buckets tend to outlive whatever they were created for. The 2024 incident (postmortem PM-112) started as a "temporary" public bucket for a campaign that later accumulated internal exports. The CDN pattern costs ~20 minutes more at setup and removes the whole failure class. This rule is a hard block rather than a judgment call because the compliant alternative is always available — there is no scenario where public-bucket is the only way.
